numpy, pandas, difflib (fuzzy title matching), TfidfVectorizer (text → vectors), cosine_similarity (vector similarity), os.

3

🧪 EDA

Check movies_data.shape, movies_data.info(), and movies_data.isna().sum(). Several text columns (especially tagline) have null values that must be handled before text processing.

4

✂️ Select 5 Features

Define selected_features = ['genres', 'keywords', 'tagline', 'cast', 'director']. These are the only columns we use — all others are ignored for the recommendation engine.

5

🧹 Fill Null Values with Empty String

Loop through each selected feature and call .fillna(''). If we left NaN values, Python would concatenate them as the literal string "nan", corrupting the TF-IDF input. Empty string means: nothing to contribute.

6

🔗 Combine Features into One String per Movie

Concatenate all 5 columns with spaces: genres + ' ' + keywords + ' ' + tagline + ' ' + cast + ' ' + director. Each movie becomes a single text document that TF-IDF can process.

7

📊 TF-IDF Vectorization

TfidfVectorizer().fit_transform(combined_features) converts all text documents into a sparse matrix of shape (4803, ~17,318) — one row per movie, one column per unique word. Each value is the TF-IDF score for that word in that movie.

8

🔁 Compute Cosine Similarity Matrix

cosine_similarity(feature_vectors) produces a 4803 × 4803 matrix. Cell [i][j] holds the similarity score between movie i and movie j — ranging from 0 (no overlap) to 1 (identical content).

9

🔎 Fuzzy Match User Input

difflib.get_close_matches(movie_name, list_of_all_titles) finds the closest real title to whatever the user types — handles typos, partial names, and capitalisation. E.g. "avtar" → "Avatar".

10

🎯 Sort & Return Top 30 Recommendations

Extract the matched movie's row from the similarity matrix, enumerate all scores, sort descending, then print the top 30 movie titles. The first result is always the movie itself (score = 1.0), followed by most similar films.

What is TF-IDF?

TF-IDF (Term Frequency–Inverse Document Frequency) scores how relevant a word is to a specific document within a collection. Common words like "the" appear everywhere and get low scores. Rare meaningful words like "Nolan" or "heist" get high scores.

Formulas
TF(t, d) = count of term t in document d ÷ total words in d
IDF(t) = 1 + loge( n ÷ df(t) )   where n = total docs, df(t) = docs containing t
TF-IDF = TF(t,d) × IDF(t)
Note: sklearn applies L2 normalization and +1 smoothing, so values differ slightly from manual calculation.

What is Cosine Similarity?

Measures the cosine of the angle between two vectors. A score of 1.0 means identical direction (movies share all the same keywords, cast, genre); 0.0 means completely unrelated. It ignores vector magnitude — a short movie description can still match a long one if they share the same terms.

Formula
cos(θ) = ( A · B ) ÷ ( |A| × |B| )
where · is dot product and |A| is the vector's magnitude (Euclidean norm)

Why do we need fuzzy matching?

Users rarely type exact movie titles. difflib.get_close_matches() from Python's standard library finds strings in a list that are most similar to the input — using the Ratcliff/Obershelp algorithm. It handles typos, partial names, and case differences.

python — complete consolidated script
import numpy as np
import pandas as pd
import difflib
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

# Load data
movies_data = pd.read_csv('movies.csv')

# Fill nulls and combine features
selected_features = ['genres', 'keywords', 'tagline', 'cast', 'director']
for feature in selected_features:
    movies_data[feature] = movies_data[feature].fillna('')

combined_features = (
    movies_data['genres'] + ' ' + movies_data['keywords'] + ' ' +
    movies_data['tagline'] + ' ' + movies_data['cast'] + ' ' +
    movies_data['director']
)

# Vectorize and compute similarity
vectorizer = TfidfVectorizer()
feature_vectors = vectorizer.fit_transform(combined_features)
similarity = cosine_similarity(feature_vectors)
list_of_all_titles = movies_data['title'].tolist()

# Get user input and recommend
movie_name = input(' Enter your favourite movie name : ')
find_close_match = difflib.get_close_matches(movie_name, list_of_all_titles)
close_match = find_close_match[0]
index_of_the_movie = movies_data[movies_data.title == close_match]['index'].values[0]
similarity_score = list(enumerate(similarity[index_of_the_movie]))
sorted_similar_movies = sorted(similarity_score, key=lambda x: x[1], reverse=True)

print(f'Movies suggested for you:\n')
i = 1
for movie in sorted_similar_movies:
    index = movie[0]
    title_from_index = movies_data[movies_data.index == index]['title'].values[0]
    if (i < 30):
        print(i, '.', title_from_index)
        i += 1

Feature Weighting

Repeat the director name 3× before combining features to increase its TF-IDF weight. Director style is often a stronger signal than individual keywords.

Easy to add

Content-Based vs Collaborative Filtering

Content-based (this project): recommends movies similar in content to ones you like — works for any user with no history needed. Limitation: can't discover movies outside your usual genres ("filter bubble").

Collaborative filtering: "users like you also liked…" — discovers diverse recommendations but requires a user-rating matrix and suffers from the cold-start problem for new users.
